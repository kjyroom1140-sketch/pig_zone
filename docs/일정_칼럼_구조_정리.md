# 일정 칼럼 구조 정리

> **환경설정 일정 마스터**(반복 스케줄)와 **일정 관리 날짜 선택**(1회성)의 칼럼 구분을 정리합니다.

---

## 1. 용도 구분

| 구분 | 용도 | 반복 여부 | 기준/기준내용 |
|------|------|-----------|---------------|
| **일정 마스터** (환경설정) | 기초 일정(작업) 정의 | 반복 스케줄 | ✅ 필요 |
| **일정 관리** (날짜 선택) | 칸·날짜 클릭 후 예정 등록 | 1회성 | ❌ 불필요 |

---

## 2. schedule_work_plans (일정 마스터 - 반복 스케줄)

**용도**: 환경설정 > 일정 마스터에서 등록하는 **반복 스케줄 기초 일정**.  
기준(criteria)과 기준내용(criteria_content)으로 **언제 반복될지** 규칙을 정의.

### 2.1 칼럼 구조

| 컬럼 | 타입 | 비고 |
|------|------|------|
| id | INTEGER | PK |
| farm_id | UUID | NULL=전역, 값 있음=농장 전용 |
| structure_template_id | INTEGER | 대상 시설 (사육시설/일반시설) |
| sortation_id | INTEGER | 구분 (schedule_sortation_definitions 또는 schedule_sortations) |
| jobtype_id | INTEGER | 작업유형 (schedule_jobtype_definitions) |
| **criteria_id** | INTEGER | **기준** (반복 규칙: 매일/요일/기간 등) |
| **criteria_content** | JSONB | **기준내용** (반복 상세: start_date, end_date, days 등) |
| work_content | TEXT | 작업내용 |
| target_structure_template_id | INTEGER | 이동 작업 시 이동 대상 시설 |
| sort_order | INTEGER | 표시 순서 |
| is_deleted | BOOLEAN | 소프트 삭제 |
| createdAt, updatedAt | TIMESTAMPTZ | |

### 2.2 일정 마스터 입력 항목

- 시설 → 구분 → 작업유형 → **기준** → **기준내용** → 작업내용
- 기준·기준내용: 반복 스케줄 규칙 (예: 출산일+21일 후 분만 예정 등)

---

## 3. schedule_executions (일정 관리 - 실행건)

**용도**: 
- (A) 반복 스케줄에서 자동 생성된 실행건 → `work_plan_id`로 연결
- (B) **날짜 선택 1회성 등록** → `work_plan_id` NULL, 구분·작업유형·작업내용 직접 저장

### 3.1 칼럼 구조 (정리안)

| 컬럼 | 타입 | 비고 |
|------|------|------|
| id | UUID | PK |
| farm_id | UUID | NOT NULL |
| **work_plan_id** | INTEGER | NULL 허용. 반복 스케줄에서 생성된 경우만 연결 |
| **sortation_id** | INTEGER | NULL 허용. 1회성 등록 시 구분 (work_plan_id NULL일 때 사용) |
| **jobtype_id** | INTEGER | NULL 허용. 1회성 등록 시 작업유형 |
| **work_content** | TEXT | NULL 허용. 1회성 등록 시 작업내용 (memo 대신) |
| section_id | UUID | NULL, 대상 칸 |
| execution_type | VARCHAR(20) | NOT NULL, 기본값 `inspection` |
| scheduled_date | DATE | NOT NULL |
| status | VARCHAR(20) | NOT NULL DEFAULT `pending` |
| completed_at | TIMESTAMPTZ | NULL |
| completed_by | UUID | NULL |
| result_ref_type | VARCHAR(30) | NULL |
| result_ref_id | UUID | NULL |
| idempotency_key | VARCHAR(80) | NULL |
| memo | TEXT | NULL (추가 메모, 선택) |
| created_at | TIMESTAMPTZ | NOT NULL |
| updated_at | TIMESTAMPTZ | NOT NULL |

### 3.2 1회성 등록(날짜 선택) 시 입력 항목

| 입력 항목 | 저장 칼럼 | 비고 |
|-----------|-----------|------|
| 대상 칸 | section_id | |
| 날짜 | scheduled_date | |
| 구분 | sortation_id | schedule_sortations.id 또는 sortation_definition_id |
| 작업유형 | jobtype_id | schedule_jobtype_definitions.id |
| 작업내용 | work_content | 직접 입력 |
| ~~기준~~ | — | **제외** |
| ~~기준내용~~ | — | **제외** |
| ~~작업 계획~~ | work_plan_id | **NULL** (1회성은 계획 미사용) |
| ~~실행 유형~~ | execution_type | 기본값 `inspection` (사용자 선택 제거) |
| 추가 메모 | memo | 선택 입력 |

### 3.3 반복 스케줄에서 생성된 실행건

| 항목 | 저장 방식 |
|------|-----------|
| work_plan_id | NOT NULL (schedule_work_plans.id) |
| sortation_id, jobtype_id, work_content | work_plan에서 JOIN으로 조회 (직접 저장 안 함) |
| criteria_id, criteria_content | work_plan에만 존재, executions에는 없음 |

## 4. 칼럼 대비표

| 구분 | 일정 마스터 (schedule_work_plans) | 일정 관리 1회성 (schedule_executions) |
|------|-----------------------------------|--------------------------------------|
| 구분 | sortation_id | sortation_id (직접) |
| 작업유형 | jobtype_id | jobtype_id (직접) |
| 작업내용 | work_content | work_content (직접) |
| **기준** | criteria_id ✅ | **없음** ❌ |
| **기준내용** | criteria_content ✅ | **없음** ❌ |
| 작업 계획 | (자신이 계획) | work_plan_id = NULL |
| 실행 유형 | — | execution_type = `inspection` (기본) |

---

## 5. 마이그레이션 요약

### 5.1 schedule_executions 변경 사항

1. **work_plan_id**: NOT NULL → **NULL 허용**
2. **sortation_id**: **추가** (INTEGER, NULL)
3. **jobtype_id**: **추가** (INTEGER, NULL)
4. **work_content**: **추가** (TEXT, NULL)
5. **execution_type**: 기본값 `inspection` 유지, 사용자 선택 UI 제거 가능

### 5.2 제약 조건

- `work_plan_id` NOT NULL 이면: 반복 스케줄에서 생성된 실행건
- `work_plan_id` NULL 이면: `sortation_id`, `jobtype_id`, `work_content` 중 하나 이상 필수 (1회성 등록)
