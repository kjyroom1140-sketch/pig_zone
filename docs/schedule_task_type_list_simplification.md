# 작업 유형 목록 간소화 방안

일정관리 설정 방법(표준 카테고리 8개)에 맞춰 **작업 유형 목록을 간소화**하는 방안입니다.

---

## 1. 표준 카테고리 (8개)

| 코드 | 표시명 | 용도 |
|------|--------|------|
| entry | 전입 | 빈방 전입, 돈군 생성 |
| move | 이동 | 이동·합사·분리 |
| vaccine | 백신·건강 | 백신, 검역, 투약 |
| feed | 사료·급여 | 사료 전환, 급여 |
| facility_management | 시설·관리 | 점검, 정리, 설비 |
| facility_environment | 시설·환경 | 환기, 온도, 습도 |
| facility_disinfection | 시설·방역 | 소독, 세척, 방역 |
| other | 기타 | 그 외 |

---

## 2. 간소화 방법 두 가지

### 2.1 표시 간소화 (권장·적용됨)

**내용**  
목록을 **카테고리별로 그룹**해 표시합니다. 같은 대분류(전입, 이동, 백신·건강 등)끼리 묶어서 보여주므로 항목이 많아도 찾기 쉽습니다.

- **admin** 일정 관리 설정 → 작업 유형 목록: 카테고리 그룹 헤더 + 그룹별 행
- **farm_admin** 농장 일정 관리 → 작업 유형 목록: 동일
- 기존 데이터는 그대로 두고, 정렬만 **카테고리 순 → sortOrder** 로 변경

**장점**  
- DB/API 변경 없음  
- 기존 18개 등 그대로 두어도 화면만 8개 블록으로 정리됨  

---

### 2.2 데이터 간소화 (선택)

**내용**  
전역·농장 작업 유형을 **표준 8개만** 두고, 나머지는 제거하거나 8개 중 하나로 통합합니다.

- **schedule_task_types**: 18개 → 8개로 축소 (같은 카테고리끼리 하나로 합치거나, 표준 8개만 남기고 나머지 삭제)
- **farm_schedule_task_types**: 농장마다 8개만 유지
- **주의**: `schedule_items`, `farm_schedule_items` 등에서 작업 유형 ID를 참조하므로, 삭제 전에 **참조를 새 ID로 갱신**하는 마이그레이션이 필요함

**진행 시**  
- 기존 “환경”, “위생” 등은 `facility_environment` 하나로 매핑  
- “예방접종”, “질병”, “관찰”, “검사”, “치료”, “판단” 등은 `vaccine` 하나로 매핑  
- 매핑 규칙을 정한 뒤 `schedule_items.taskTypeId`, `farm_schedule_items.taskTypeId` 등을 새 ID로 UPDATE하고, 중복 작업 유형 행 삭제  

데이터 간소화를 원하면 별도 마이그레이션 스크립트 설계가 필요합니다.

---

## 3. 정리

| 구분 | 내용 |
|------|------|
| **현재 적용** | **표시 간소화**: 작업 유형 목록을 카테고리(전입, 이동, 백신·건강 등 8개)별로 그룹해 표시 |
| **선택 사항** | **데이터 간소화**: 실제 행을 8개로 줄이려면 참조 갱신 마이그레이션 필요 |

일정관리 설정 방법에 맞추려면 **표준 8개 카테고리**만 쓰고, 추가·수정 시에도 이 카테고리 셀렉트만 사용하는 것이 좋습니다.  
목록은 “표시 간소화”로 이미 카테고리 기준으로 정리되어 있습니다.
